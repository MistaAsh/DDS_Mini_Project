PROCEDURE

1. Building the full subtractor model

1. First, we start with creating a .v file in the 'Implementaton' section of the Xilinx software. In the course of this project, we named it 'full_subtractor.v'. This file will contain the module which builds the basic framework of the project.

2. We define the inputs using the 'inputs' datatype and which are- 
                  A - The minuend
                  B - The subtrahend
                  Bin - The borrow-in

3. Next we define the outputs under the same module, which for a full-subtractor is-
                  D - The Difference
                  Bout - The borrow-out

4. Finally, after building the module signature, we top it off by initializing each output as a function of its inputs. In the case of a full-subtractor they are-           
                  D = A^B^Bin
                  Bout = A'B + A'Bin + BBin

2. Creating testcases for simulations

1. To see the difference in the actual practical outputs (i.e. inputs which can be stuck-at 0) with the expected outputs, we create a set a inputs to represent the ideal case where no input can have a logical error and subsequently get the ideal, expected output for the same. The ideal inputs/outputs created for the subtractor are-
                  _A - The ideal case Minuend (input)
                  _B - The ideal case Subtrahend(input)
                  _Bin- The ideal case borrow-in (input)
                  _D - The ideal case difference (output)
                  _Bout- the ideal case borrow-out (output)

2. We also create a separate variable 'flag' which is initialized to 0 representing that no faults have been detected yet. This flag is later assigned 1 if a fault is detected within the circuit.

3. Running the simulations

1. For different values of inputs provided, we check the values obtained as output.

2. The output values will be unique in case of some of the faults, using which we can identify where the stuck-at 0 fault is located.

3. We check the values of the outputs in the given order and when a fault is found, we update the value of flag to 1, print the location of the fault and end module.

When A=1, B=1, Bin=1: 
            * If D=0 & Bout= 1, then A is stuck-at 0            
            * If D = 1 & Bout = 0, then both B and Bin is stuck-at 0
When  A = 1, B=1, Bin=0: 
           * If D= 1 & Bout= 0, then B is stuck-at 0            
           * If D = 1 & Bout = 1, then both A and Bin is stuck-at 0
When A= 1, B=0, Bin=0
           * If D=1 & Bout= 0, then Bin is stuck-at 0           
           * If D=1 & Bout= 1, then both A and B is stuck-at 0
When A=1, B=0, Bin=0
           * If D=0 & Bout= 0, then A,B and Bin is stuck-at 0
